FAQs

How does it handle my data?

The toolbox is open source, free to use, and runs locally inside ArcGIS Pro. Only the context you choose for a tool is sent to the model endpoint you configure, and you can inspect or modify every line of code if you want to tighten things further.

How do I get an OpenRouter API key?

  1. Create an OpenRouter account at openrouter.ai
  2. Head to Settings > API Keys
  3. Click Create Key, give it a descriptive name (like "ArcGIS Pro AI"), and copy the secret value
  4. Use setx to store the API key on your machine:
    setx OPENROUTER_API_KEY "your-openrouter-key"
  5. (Optional) Add credit on the Billing page or attach a card so requests can run without limits
Checking usage & cost
  • Review recent calls and spending on the OpenRouter Activity page.
  • Mini models on OpenRouter are inexpensive. Typical runs cost under $0.01.

Why does this use OpenRouter?

OpenRouter gives one transparent API for many language models, so the toolbox can stay model-agnostic.

  • Pick models based on cost, speed, or quality without rewriting tools.
  • Switch providers without changing code or workflows.
  • Avoid lock-in by bringing whichever API key fits your project.

Prefer another provider? The code is open and designed to be adapted.

Who made this?

This toolbox was built by a GIS analyst who likes building things.

I’m a former Esri employee and long-time ArcGIS Pro user. I’ve worked on real GIS teams, maintained production workflows, and felt the friction when powerful tools become rigid. Lately, I’ve been exploring how AI can act as a thinking partner inside GIS workflows, not a replacement for judgment.

This toolbox reflects that perspective: practical, inspectable, and built by someone who actually uses ArcGIS Pro every day.

Why isn’t this a product?

Because open source is better for this stage.

Keeping this open source lets anyone use it, inspect it, and adapt it. It also lets me build honestly, without optimizing for pricing tiers or feature checklists. If it grows into something people want to support later, great. If it stays a lightweight tool that quietly improves workflows, that’s great too.

For now, the goal is simple: let it become what it should be.

Is AI actually right for GIS work?

It can be, if you use it thoughtfully.

AI can hallucinate and get things wrong. That’s real. But GIS work has always required skepticism, verification, and judgment. This toolbox is designed for analysts who already work that way.

Used responsibly, AI can reduce friction, summarize complex context, surface ideas worth investigating, and act as a second set of eyes, not an authority. This is about supporting human-led analysis, not automating GIS.

Is AI going to take my GIS job?

No. It changes the work, not the need for judgment.

AI is good at generating text and suggestions. It is not good at understanding context, responsibility, or real-world consequences. GIS work lives in the real world, where decisions matter.

What will change is speed. Analysts who benefit most from AI already understand their data, know what questions matter, can spot bad answers, and take responsibility for results. This toolbox assumes GIS analysts are thinking professionals, not button-pushers.

If a job can be fully replaced by a prompt, it probably wasn’t really GIS work to begin with.
